#### C. 登录失败锁定（安全增强）
新增机制：
- 连续失败阈值：`5次`
- 锁定时长：`10分钟`
- 字段落库：
    - `login_fail_count`
    - `lock_expire_time`

登录逻辑：
1. 先判断是否在锁定期。
2. 密码错误 -> 失败次数+1；第5次触发锁定。
3. 密码正确 -> 清失败次数、清锁定时间、更新登录时间。

作用：  
提升账号安全性，防爆破，属于面试高价值安全点。

### 3. 关键问题与解答（Q&A）

Q1：为什么 `ApiException("字符串")` 报错？  
A：你的 `ApiException` 构造器接收的是 `IErrorCode`，不是 `String`。应新增枚举错误码后抛 `new ApiException(UmsErrorCode.xxx)`。

Q2：为什么锁定后登录成功，`lock_expire_time` 还没清空？  
A：`updateById` 在默认策略下对 `null` 字段可能不更新。  
解决：改用 `LambdaUpdateWrapper` 显式 `set(lockExpireTime, null)`。

Q3：为什么明明刚登录拿 token，还 401？  
A：你已接入 Redis 黑名单，若 Redis 未启动或 token 在黑名单/过期，会返回 401。你本次已确认是 Redis 未启动导致。

Q4：角色资源变更后，为什么不用重新登录就生效？  
A：token 只承载身份，权限是每次请求动态加载；你又做了权限缓存自动失效，所以同 token 可立即看到新权限。
